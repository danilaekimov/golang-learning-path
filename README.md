Блок 1: Основы языка

Теория: Синтаксис, переменные, базовые типы данных (int, string, bool), константы, операторы, управляющие конструкции (if/else, switch, for).​

Дополнить код: Упражнения на написание функций с использованием циклов и ветвлений.

Самостоятельное задание: Написать программу "Консольный калькулятор", которая принимает два числа и оператор, а затем выводит результат.​

Реальный проект: Начать CLI-утилиту (инструмент командной строки). Калькулятор может стать ее первой командой.

Блок 2: Составные типы данных

Теория: Массивы, срезы (slices), карты (maps), структуры (structs). Понимание разницы между срезами и массивами.​

Дополнить код: Написать функции для сортировки среза, поиска элемента в карте.

Самостоятельное задание: Создать программу для ведения списка дел (To-Do List) в памяти. Задачи можно хранить в срезе структур.​

Реальный проект: Расширить CLI-утилиту, добавив возможность управлять списком задач (добавить, удалить, показать).

Блок 3: Функции, указатели и обработка ошибок

Теория: Объявление функций, множественные возвращаемые значения, именованные результаты, указатели, стандартный механизм обработки ошибок в Go через возвращение error.​

Дополнить код: Рефакторинг предыдущих заданий с вынесением логики в отдельные функции и добавлением обработки ошибок.

Самостоятельное задание: Написать функцию, которая делит два числа и корректно обрабатывает ошибку деления на ноль.

Реальный проект: Добавить в To-Do List сохранение и чтение задач из файла (например, в формате JSON), реализовав корректную обработку ошибок при работе с файлами.

Блок 4: Пакеты, модули и тестирование

Теория: Организация кода в пакеты, импорт и экспорт, управление зависимостями с помощью Go Modules (go.mod), основы тестирования с помощью встроенного пакета testing.

Дополнить код: Разнести код To-Do листа по разным пакетам (например, internal/tasks и cmd/todo-cli).

Самостоятельное задание: Написать unit-тесты для функций вашего калькулятора и менеджера задач.

Реальный проект: Превратить To-Do лист в полноценное приложение, которое можно собрать и запустить, с тестами, покрывающими основную логику.

Блок 5: Конкурентность (ключевая особенность Go)

Теория: Горутины (goroutines) и каналы (channels) — основа конкурентного программирования в Go. WaitGroup для синхронизации.​

Дополнить код: Упражнения на запуск нескольких горутин и обмен данными через каналы.

Самостоятельное задание: Написать программу, которая конкурентно скачивает содержимое нескольких веб-страниц и выводит их заголовки.

Реальный проект: Создать веб-парсер (Web Scraper). Программа должна принимать список URL, конкурентно их обходить и извлекать нужную информацию (например, заголовки статей).​

Блок 6: Разработка API

Теория: Стандартная библиотека net/http для создания веб-серверов. Обработка HTTP-запросов, маршрутизация, работа с JSON.​

Дополнить код: Создать простой HTTP-сервер, который отвечает "Hello, World!".

Самостоятельное задание: Написать простое REST API для вашего To-Do листа (создание, получение, удаление задач через HTTP-запросы).

Реальный проект: Разработать сервер для хранения заметок или простой чат-бот для Telegram, который будет общаться с Telegram API.​

Последующие блоки могут включать:

Блок 7: Работа с базами данных (SQL и NoSQL).

Блок 8: Контекст (Context) и его использование в API и конкурентных программах.

Блок 9: Продвинутые темы: рефлексия, "чистая" архитектура, DDD.​

Блок 10: Создание полноценного сервиса, например, мониторинга доступности сайтов с уведомлениями или сервиса, похожего на Gogs (упрощенный Git-сервис).
